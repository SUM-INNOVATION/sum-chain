## Abstract

SRC-201 defines a standard for on-chain encrypted messaging on SUM Chain. Every message is a blockchain transaction, ensuring full decentralization with no relay servers required. The standard includes a Registry Contract that sponsors message gas fees, enabling free messaging during the adoption phase with a path to paid messaging in the future.

**Key Design Decisions:**
- **Registry-as-recipient pattern** for metadata privacy (recipient address hidden)
- **AEAD with explicit AAD** for tamper-proof headers
- **Structured attachment format** for interoperability
- **Anti-spam mechanisms** beyond simple rate limiting

## 1. Motivation

### Problems with Centralized Messaging
- Single point of failure
- Censorship risk
- Privacy concerns (server can read metadata)
- Requires trust in operator

### SRC-201 Solution
- **100% Decentralized**: Messages stored on-chain
- **Censorship Resistant**: No entity can block messages
- **Cryptographic Privacy**: End-to-end encryption
- **Metadata Privacy**: Registry-as-recipient hides social graph
- **Blockchain Finality**: Delivery guaranteed by consensus
- **Native Payments**: Attach Koppa to any message

### Core Principles

| Principle | Description |
|-----------|-------------|
| Messages are Transactions | Every message is a valid SUM Chain transaction |
| Calldata Storage | Message content in tx `data` field (NOT contract state) |
| Event-based Indexing | Registry emits events for message discovery |
| Metadata Privacy | Recipient hidden via registry-as-recipient pattern |
| Immutable | Once confirmed, messages cannot be deleted |
| Self-Sovereign | Users control their own keys, no accounts needed |

### 3.2 Registry-as-Recipient Pattern

**Why this design:**
- Direct-to-recipient (`to = recipient`) leaks social graph permanently
- With registry-as-recipient, all messages go to the same contract address
- Recipient address is encrypted inside the payload
- Recipients scan registry events/logs to find their messages

**How recipients discover messages:**
1. Registry emits `MessageDeposited(sender, recipientHash, messageId)` event
2. `recipientHash = BLAKE3(recipientAddress)` - one-way hash
3. Recipient computes their own hash and filters events
4. Full recipient address never appears on-chain in plaintext

```rust
// Event structure
event MessageDeposited {
    sender: Address,              // Visible (unavoidable)
    recipient_hash: [u8; 32],     // BLAKE3 hash of recipient address
    message_id: Hash,             // Transaction hash
    size: u32,                    // Message size in bytes
    timestamp: u64,               // Block timestamp
}
```

## 4. Registry Contract

The Registry Contract is the core infrastructure for SRC-201. It handles gas sponsorship, rate limiting, anti-spam, and event emission.

### 4.1 Contract Address (Reserved)

```
Mainnet:  sum1s201reg0000000000000000000000000001
Testnet:  sumtest1s201reg000000000000000000000001

Magic bytes: 0x53323031 ("S201" in ASCII)
```

### 4.2 Contract Interface

```rust
// SRC-201 Registry Contract Interface
// Language: Rust (for SUM Chain VM)

/// Registry contract for SRC-201 messaging
pub trait SRC201Registry {

    // ═══════════════════════════════════════════════════════════════════════
    // CORE MESSAGING FUNCTIONS
    // ═══════════════════════════════════════════════════════════════════════

    /// Send a message with gas sponsorship (meta-transaction)
    ///
    /// # Arguments
    /// * `message_data` - Encoded SRC-201 message (encrypted, contains recipient)
    /// * `recipient_hash` - BLAKE3 hash of recipient address (for indexing)
    /// * `signature` - Sender's signature over the message envelope
    /// * `nonce` - Sender's message nonce (prevents replay)
    /// * `expiry` - Unix timestamp when this meta-tx expires
    ///
    /// # Returns
    /// * `message_id` - Transaction hash (unique identifier)
    fn send_message(
        message_data: Vec<u8>,
        recipient_hash: [u8; 32],
        signature: Signature,
        nonce: u64,
        expiry: u64,
    ) -> Result<Hash, Error>;

    /// Send a message with attached Koppa payment
    /// Payment is held in escrow until recipient claims
    fn send_message_with_payment(
        message_data: Vec<u8>,
        recipient_hash: [u8; 32],
        koppa_amount: u128,
        signature: Signature,
        nonce: u64,
        expiry: u64,
    ) -> Result<Hash, Error>;

    /// Direct send (user pays own gas, no sponsorship)
    /// Still goes through registry for consistent event emission
    fn send_message_direct(
        message_data: Vec<u8>,
        recipient_hash: [u8; 32],
    ) -> Result<Hash, Error>;

    /// Claim payment from a message (called by recipient)
    fn claim_payment(
        message_id: Hash,
        recipient_proof: RecipientProof,
    ) -> Result<(), Error>;

    // ═══════════════════════════════════════════════════════════════════════
    // RATE LIMITING & ANTI-SPAM
    // ═══════════════════════════════════════════════════════════════════════

    /// Get remaining free messages for an address today
    fn get_daily_quota(sender: Address) -> QuotaInfo;

    /// Check if address is currently rate-limited
    fn is_rate_limited(sender: Address) -> bool;

    /// Get sender's spam score (higher = more restricted)
    fn get_spam_score(sender: Address) -> u32;

    /// Stake Koppa to increase trust level (anti-spam bond)
    fn stake_for_trust(amount: u128) -> Result<(), Error>;

    /// Withdraw stake (with cooldown period)
    fn unstake(amount: u128) -> Result<(), Error>;

    // ═══════════════════════════════════════════════════════════════════════
    // RECIPIENT CONTROLS
    // ═══════════════════════════════════════════════════════════════════════

    /// Set filter mode for incoming messages
    /// 0 = accept all, 1 = contacts only, 2 = staked senders only
    fn set_inbox_filter(mode: u8) -> Result<(), Error>;

    /// Add address to contacts (allowed senders)
    fn add_contact(contact_hash: [u8; 32]) -> Result<(), Error>;

    /// Remove address from contacts
    fn remove_contact(contact_hash: [u8; 32]) -> Result<(), Error>;

    /// Block a sender (messages still on-chain but marked blocked)
    fn block_sender(sender: Address) -> Result<(), Error>;

    // ═══════════════════════════════════════════════════════════════════════
    // ADMIN FUNCTIONS (Governance controlled)
    // ═══════════════════════════════════════════════════════════════════════

    /// Set daily free message quota (default: 100)
    fn set_daily_quota(quota: u32) -> Result<(), Error>;

    /// Set message size limit in bytes (default: 4096)
    fn set_max_message_size(size: u32) -> Result<(), Error>;

    /// Set minimum stake for "trusted sender" tier
    fn set_min_trust_stake(amount: u128) -> Result<(), Error>;

    /// Pause/unpause sponsorship (emergency)
    fn set_sponsorship_enabled(enabled: bool) -> Result<(), Error>;

    /// Fund the registry with Koppa for sponsorship
    fn fund_registry() -> Result<(), Error>;  // payable
}

/// Quota information for a sender
pub struct QuotaInfo {
    pub used_today: u32,
    pub remaining: u32,
    pub total_quota: u32,
    pub tier: u8,
    pub stake_amount: u128,
    pub resets_at: u64,  // Unix timestamp
}
```

### 5.1 Message Transaction Structure

```
SRC-201 Message Transaction:
{
    // Standard transaction fields
    "from": "<sender_sum_address>",
    "to": "<REGISTRY_CONTRACT_ADDRESS>",    // Always to registry!

```

### 5.2 Binary Encoding Format (v1.1)

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                   SRC-201 MESSAGE BINARY FORMAT (v1.1)                       │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                              │
│  HEADER (fixed 72 bytes) - included in AEAD AAD                             │
│  ───────────────────────────────────────────────────                        │
│  Offset │ Size    │ Field              │ Description                        │
│  ───────┼─────────┼────────────────────┼──────────────────────────────────  │
│  0      │ 4 bytes │ Magic              │ 0x53323031 ("S201" in ASCII)       │
│  4      │ 1 byte  │ Version            │ Protocol version (currently 1)     │
│  5      │ 1 byte  │ Flags              │ Bit flags (see below)              │
│  6      │ 1 byte  │ Content Type       │ MIME type enum                     │
│  7      │ 1 byte  │ Attachment Count   │ Number of attachments (0-255)      │
│  8      │ 32 bytes│ Recipient Hash     │ BLAKE3(recipient_address)          │
│  40     │ 32 bytes│ Ephemeral PubKey   │ X25519 public key for ECDH         │
│                                                                              │
│  ENCRYPTION PARAMS (fixed 26 bytes) - NOT in AAD                            │
│  ──────────────────────────────────────────────────                         │
│  72     │ 24 bytes│ Nonce              │ XChaCha20-Poly1305 nonce           │
│  96     │ 2 bytes │ Payload Length     │ uint16 big-endian (max 65535)      │
│                                                                              │
│  ENCRYPTED PAYLOAD (variable)                                                │
│  ─────────────────────────────                                              │
│  98     │ N bytes │ Ciphertext         │ Encrypted: recipient + content     │
│  98+N   │ 16 bytes│ Auth Tag           │ Poly1305 tag (covers header+payload)│
│                                                                              │
│  OPTIONAL FIELDS (based on flags)                                            │
│  ─────────────────────────────────                                          │
│  114+N  │ 32 bytes│ Reply-To (opt)     │ Parent message tx hash             │
│  146+N  │ 8 bytes │ Timestamp (opt)    │ Sender-provided Unix timestamp     │
│  154+N  │ var     │ Attachments (opt)  │ Attachment records (see section 7) │
│                                                                              │
│  Minimum size: 114 bytes (no attachments, no optional fields)               │
│  Maximum size: 65535 bytes (limited by payload length field)                │
│                                                                              │
└─────────────────────────────────────────────────────────────────────────────┘
```

### 5.3 Flag Byte Definition

```
Flags Byte (offset 5):
┌───┬───┬───┬───┬───┬───┬───┬───┐
│ 7 │ 6 │ 5 │ 4 │ 3 │ 2 │ 1 │ 0 │
└───┴───┴───┴───┴───┴───┴───┴───┘
  │   │   │   │   │   │   │   │
  │   │   │   │   │   │   │   └── [0] Encrypted (1) / Public (0)
  │   │   │   │   │   │   └────── [1] Has Reply-To field (32 bytes)
  │   │   │   │   │   └────────── [2] Has Timestamp field (8 bytes)
  │   │   │   │   └────────────── [3] Has Attachments (see attachment_count)
  │   │   │   └────────────────── [4] Is Read Receipt
  │   │   └────────────────────── [5] Is Payment Request
  │   └────────────────────────── [6] Requires Stake (anti-spam)
  └────────────────────────────── [7] Reserved (must be 0)

Common Flag Values:
  0x01 = Encrypted direct message
  0x03 = Encrypted message with reply reference
  0x07 = Encrypted message with reply + timestamp
  0x0F = Encrypted message with reply + timestamp + attachments
  0x20 = Payment request
  0x10 = Read receipt
  0x00 = Public broadcast (unencrypted)
```


## 6. Encryption Scheme

### 6.1 Algorithm Selection

```
Encryption: X25519-XChaCha20-Poly1305 with explicit AAD

┌─────────────────────────────────────────────────────────────────────────────┐
│ Component        │ Algorithm              │ Reason                          │
├──────────────────┼────────────────────────┼─────────────────────────────────┤
│ Key Exchange     │ X25519 (Curve25519)    │ Fast, secure, compact keys     │
│ Symmetric Cipher │ XChaCha20              │ 24-byte nonce (safe random)    │
│ Authentication   │ Poly1305               │ Fast MAC, AEAD construction    │
│ KDF              │ BLAKE3                 │ Fast, matches SUM Chain hash   │
│ Recipient Hash   │ BLAKE3                 │ One-way, 32 bytes              │
└─────────────────────────────────────────────────────────────────────────────┘
```

### 6.2 AEAD Associated Data (Critical for Tamper Protection)

**All header fields are authenticated but not encrypted.**

The AEAD `additional_data` parameter MUST include the entire header to prevent attackers from modifying flags, content type, or other header fields.

```
AAD (Associated Authenticated Data) = bytes[0:72] of the message

Specifically:
  AAD = magic (4) || version (1) || flags (1) || content_type (1) ||
        attachment_count (1) || recipient_hash (32) || ephemeral_pubkey (32)

Total AAD size: 72 bytes (fixed)

The AAD is:
  - NOT encrypted (visible on-chain)
  - Authenticated by Poly1305 tag
  - Any modification will cause decryption to fail
```

### 6.3 Encryption Process (Sender)

```
ENCRYPTION FLOW:

1. GENERATE EPHEMERAL KEYPAIR
   ┌─────────────────────────────────────────────────────────────┐
   │ ephemeralPrivate = random(32 bytes)                         │
   │ ephemeralPublic = X25519.publicKey(ephemeralPrivate)        │
   └─────────────────────────────────────────────────────────────┘

2. CONVERT RECIPIENT ED25519 KEY TO X25519
   ┌─────────────────────────────────────────────────────────────┐
   │ recipientX25519 = ed25519_pk_to_curve25519(recipientEdPubKey)│
   │ // Use libsodium: crypto_sign_ed25519_pk_to_curve25519()    │
   └─────────────────────────────────────────────────────────────┘

3. DERIVE SHARED SECRET (ECDH)
   ┌─────────────────────────────────────────────────────────────┐
   │ sharedPoint = X25519(ephemeralPrivate, recipientX25519)     │
   └─────────────────────────────────────────────────────────────┘

4. KEY DERIVATION (BLAKE3)
   ┌─────────────────────────────────────────────────────────────┐
   │ context = "SRC-201-v1.1-message-key"                        │
   │ messageKey = BLAKE3.deriveKey(context, sharedPoint)         │
   │ // Output: 32 bytes                                         │
   └─────────────────────────────────────────────────────────────┘

5. BUILD PLAINTEXT
   ┌─────────────────────────────────────────────────────────────┐
   │ plaintext = recipientAddress (32) ||                        │
   │             recipientEdPubKey (32) ||                        │
   │             messageContent (variable)                        │
   └─────────────────────────────────────────────────────────────┘

6. BUILD HEADER (this becomes AAD)
   ┌─────────────────────────────────────────────────────────────┐
   │ recipientHash = BLAKE3(recipientAddress)                    │
   │ header = magic || version || flags || contentType ||        │
   │          attachmentCount || recipientHash || ephemeralPublic│
   │ // header is exactly 72 bytes                               │
   └─────────────────────────────────────────────────────────────┘

7. GENERATE NONCE
   ┌─────────────────────────────────────────────────────────────┐
   │ nonce = random(24 bytes)                                    │
   │ // XChaCha20's 24-byte nonce is safe for random generation  │
   └─────────────────────────────────────────────────────────────┘

8. ENCRYPT WITH AEAD
   ┌─────────────────────────────────────────────────────────────┐
   │ (ciphertext, tag) = XChaCha20Poly1305.encrypt(              │
   │     key: messageKey,                                         │
   │     nonce: nonce,                                            │
   │     plaintext: plaintext,                                    │
   │     aad: header          // CRITICAL: authenticate header   │
   │ )                                                            │
   └─────────────────────────────────────────────────────────────┘

9. ASSEMBLE MESSAGE
   ┌─────────────────────────────────────────────────────────────┐
   │ message = header || nonce || payloadLength || ciphertext || │
   │           tag || [replyTo] || [timestamp] || [attachments]  │
   └─────────────────────────────────────────────────────────────┘

10. CLEAR SENSITIVE DATA
    ┌─────────────────────────────────────────────────────────────┐
    │ secureZero(ephemeralPrivate)                                │
    │ secureZero(sharedPoint)                                     │
    │ secureZero(messageKey)                                      │
    │ secureZero(plaintext)                                       │
    └─────────────────────────────────────────────────────────────┘
```

### 6.4 Decryption Process (Recipient)

```
DECRYPTION FLOW:

1. PARSE MESSAGE
   ┌─────────────────────────────────────────────────────────────┐
   │ Verify magic == 0x53323031 ("S201")                         │
   │ Extract header (bytes 0-71) as AAD                          │
   │ Extract: nonce, payloadLength, ciphertext, tag              │
   └─────────────────────────────────────────────────────────────┘

2. VERIFY RECIPIENT HASH (optional fast-reject)
   ┌─────────────────────────────────────────────────────────────┐
   │ myHash = BLAKE3(myAddress)                                  │
   │ if header.recipientHash != myHash:                          │
   │     SKIP (message not for us)                               │
   └─────────────────────────────────────────────────────────────┘

3. CONVERT MY ED25519 KEY TO X25519
   ┌─────────────────────────────────────────────────────────────┐
   │ myX25519Private = ed25519_sk_to_curve25519(myEdPrivateKey)  │
   │ // Use libsodium: crypto_sign_ed25519_sk_to_curve25519()    │
   └─────────────────────────────────────────────────────────────┘

4. DERIVE SHARED SECRET (ECDH)
   ┌─────────────────────────────────────────────────────────────┐
   │ ephemeralPublic = header.ephemeralPubKey                    │
   │ sharedPoint = X25519(myX25519Private, ephemeralPublic)      │
   └─────────────────────────────────────────────────────────────┘

5. KEY DERIVATION (BLAKE3)
   ┌─────────────────────────────────────────────────────────────┐
   │ context = "SRC-201-v1.1-message-key"                        │
   │ messageKey = BLAKE3.deriveKey(context, sharedPoint)         │
   └─────────────────────────────────────────────────────────────┘

6. DECRYPT WITH AEAD
   ┌─────────────────────────────────────────────────────────────┐
   │ plaintext = XChaCha20Poly1305.decrypt(                      │
   │     key: messageKey,                                         │
   │     nonce: nonce,                                            │
   │     ciphertext: ciphertext,                                  │
   │     tag: tag,                                                │
   │     aad: header          // MUST match encryption AAD       │
   │ )                                                            │
   │                                                              │
   │ IF decryption fails:                                         │
   │     REJECT (tampering, wrong recipient, or corruption)      │
   │     // IMPORTANT: same error for all failure modes          │
   └─────────────────────────────────────────────────────────────┘

7. PARSE DECRYPTED PAYLOAD
   ┌─────────────────────────────────────────────────────────────┐
   │ recipientAddress = plaintext[0:32]                          │
   │ recipientPubKey = plaintext[32:64]                          │
   │ messageContent = plaintext[64:]                             │
   │                                                              │
   │ // Verify we are the intended recipient                     │
   │ if recipientAddress != myAddress:                           │
   │     REJECT (should not happen if hash matched)              │
   └─────────────────────────────────────────────────────────────┘

8. CLEAR SENSITIVE DATA
   ┌─────────────────────────────────────────────────────────────┐
   │ secureZero(myX25519Private)                                 │
   │ secureZero(sharedPoint)                                     │
   │ secureZero(messageKey)                                      │
   └─────────────────────────────────────────────────────────────┘
```

### 6.5 Ed25519 to X25519 Key Conversion

```
SUM Chain uses Ed25519 keys for signing, but X25519 is needed for encryption.
This conversion is well-defined and secure.

┌─────────────────────────────────────────────────────────────────────────────┐
│                     ED25519 TO X25519 CONVERSION                             │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                              │
│  For Private Keys (Ed25519 seed → X25519 private):                          │
│  ────────────────────────────────────────────────                           │
│  Use libsodium: crypto_sign_ed25519_sk_to_curve25519(x25519_sk, ed25519_sk) │
│                                                                              │
│  Manual process (if not using libsodium):                                   │
│  1. Take Ed25519 private key seed (32 bytes)                                │
│  2. Hash with SHA-512: h = SHA512(seed)                                     │
│  3. Take first 32 bytes and clamp:                                          │
│     x25519_private = h[0:32]                                                │
│     x25519_private[0] &= 248                                                │
│     x25519_private[31] &= 127                                               │
│     x25519_private[31] |= 64                                                │
│                                                                              │
│  For Public Keys (Ed25519 pub → X25519 pub):                                │
│  ──────────────────────────────────────────                                 │
│  Use libsodium: crypto_sign_ed25519_pk_to_curve25519(x25519_pk, ed25519_pk) │
│                                                                              │
│  Manual process:                                                             │
│  1. Ed25519 public key is a point on Edwards curve (y coordinate)           │
│  2. Convert to Montgomery form using birational map:                        │
│     u = (1 + y) / (1 - y) mod p                                            │
│  3. Result is X25519 public key (u coordinate)                             │
│                                                                              │
│  RECOMMENDED: Use libsodium for these conversions.                          │
│                                                                              │
└─────────────────────────────────────────────────────────────────────────────┘
```


### 7.2 Attachment Encryption

Each inline attachment is encrypted separately with a derived key:

```
For attachment index i (0-based):
  attachment_key = BLAKE3.deriveKey(
      context: "SRC-201-v1.1-attachment-key",
      input: messageKey || i (as uint8)
  )

  (ciphertext, tag) = XChaCha20Poly1305.encrypt(
      key: attachment_key,
      nonce: nonce,  // Same nonce as main message (safe: different key)
      plaintext: attachment_data,
      aad: attachment_hash  // Bind encryption to expected content
  )
```

## 8. Anti-Spam Mechanisms

### 8.1 Multi-Layer Protection

Simple per-address rate limiting is insufficient because spammers can create unlimited addresses. SRC-201 implements multiple layers:

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                       ANTI-SPAM LAYERS                                       │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                              │
│  LAYER 1: Rate Limiting (Basic)                                             │
│  ────────────────────────────────                                           │
│  • 100 free messages/day per address                                        │
│  • Quota resets at midnight UTC                                             │
│  • Easily bypassed by creating new addresses                                │
│                                                                              │
│  LAYER 2: Stake Requirement (Sybil Resistance)                              │
│  ─────────────────────────────────────────────                              │
│  • Stake 100+ Koppa to unlock "trusted sender" tier                         │
│  • Trusted senders: 1000 messages/day, lower spam score                     │
│  • Stake is slashed if reported as spam by multiple recipients              │
│  • Stake can be withdrawn after 7-day cooldown                              │
│  • Economic cost makes mass spam expensive                                  │
│                                                                              │
│  LAYER 3: Recipient Filtering (User Control)                                │
│  ──────────────────────────────────────────────                             │
│  • Recipients set inbox_filter mode:                                        │
│    - Mode 0: Accept all (default)                                           │
│    - Mode 1: Contacts only (must be in contact list)                        │
│    - Mode 2: Staked senders only (require stake)                            │
│  • Recipients can block specific senders                                    │
│  • Blocked messages still on-chain but filtered client-side                 │
│                                                                              │
│  LAYER 4: Spam Scoring (Reputation)                                         │
│  ───────────────────────────────────                                        │
│  • Each address has spam_score (0-100)                                      │
│  • Score increases when:                                                    │
│    - Reported as spam by recipients                                         │
│    - High message volume with low reply rate                                │
│    - Many messages to addresses that block sender                           │
│  • Score decreases over time (decay) and with successful interactions       │
│  • High spam score (>50): restricted to 10 messages/day                     │
│  • Very high spam score (>80): requires stake to send                       │
│                                                                              │
│  LAYER 5: Optional Proof-of-Work (Future)                                   │
│  ──────────────────────────────────────────                                 │
│  • Senders can include a PoW stamp in message                               │
│  • Adds small computational cost (1-5 seconds)                              │
│  • Cheap for humans, expensive at scale for spammers                        │
│  • Recipients can require PoW for unknown senders                           │
│                                                                              │
└─────────────────────────────────────────────────────────────────────────────┘
```

### 8.2 Stake Slashing Mechanism

```rust
/// Report a sender as spam
/// If enough reports, sender's stake is slashed
fn report_spam(
    sender: Address,
    message_id: Hash,
    evidence: SpamEvidence,
) -> Result<(), Error> {
    // Require reporter to also have stake (prevent griefing)
    require(stake_balances[caller] >= MIN_REPORTER_STAKE);

    // Record report
    spam_reports[message_id].push(SpamReport {
        reporter: caller,
        timestamp: now(),
        evidence,
    });

    // If 3+ unique reporters, increase spam score
    if spam_reports[message_id].len() >= 3 {
        spam_scores[sender] += 10;

        // If spam score > 80 and sender has stake, slash 10%
        if spam_scores[sender] > 80 && stake_balances[sender] > 0 {
            let slash_amount = stake_balances[sender] / 10;
            stake_balances[sender] -= slash_amount;
            // Slashed funds go to sponsorship pool
            sponsorship_balance += slash_amount;
        }
    }
}
```

### 10.2 Client Implementation (Swift/iOS)

```swift
// File: SUMWallet/Modules/Services/Sources/SRC201Service.swift

import Foundation
import CryptoKit

// MARK: - SRC-201 Constants

public enum SRC201Constants {
    static let magic: [UInt8] = [0x53, 0x32, 0x30, 0x31]  // "S201"
    static let version: UInt8 = 1
    static let maxMessageSize: Int = 65535
    static let nonceSize: Int = 24
    static let tagSize: Int = 16
    static let publicKeySize: Int = 32
    static let hashSize: Int = 32
    static let headerSize: Int = 72
    static let kdfContext = "SRC-201-v1.1-message-key"
    static let attachmentKdfContext = "SRC-201-v1.1-attachment-key"
}

// MARK: - Message Flags

public struct SRC201Flags: OptionSet {
    public let rawValue: UInt8

    public static let encrypted       = SRC201Flags(rawValue: 1 << 0)
    public static let hasReplyTo      = SRC201Flags(rawValue: 1 << 1)
    public static let hasTimestamp    = SRC201Flags(rawValue: 1 << 2)
    public static let hasAttachments  = SRC201Flags(rawValue: 1 << 3)
    public static let isReadReceipt   = SRC201Flags(rawValue: 1 << 4)
    public static let isPaymentReq    = SRC201Flags(rawValue: 1 << 5)
    public static let requiresStake   = SRC201Flags(rawValue: 1 << 6)

    public init(rawValue: UInt8) {
        self.rawValue = rawValue
    }
}

// MARK: - Content Types

public enum SRC201ContentType: UInt8 {
    case textPlain = 0x01
    case textMarkdown = 0x02
    case applicationJson = 0x10
    case paymentRequest = 0x80
    case readReceipt = 0x81
    case custom = 0xFF
}

// MARK: - SRC-201 Message

public struct SRC201Message {
    // Header (72 bytes, used as AAD)
    public let magic: [UInt8]
    public let version: UInt8
    public let flags: SRC201Flags
    public let contentType: SRC201ContentType
    public let attachmentCount: UInt8
    public let recipientHash: Data      // 32 bytes
    public let ephemeralPublicKey: Data // 32 bytes

    // Encryption params
    public let nonce: Data              // 24 bytes
    public let encryptedPayload: Data   // Variable
    public let authTag: Data            // 16 bytes

    // Optional fields
    public let replyTo: String?
    public let timestamp: UInt64?
    public let attachments: [SRC201Attachment]

    /// Build header bytes for AAD
    public var headerBytes: Data {
        var data = Data(capacity: SRC201Constants.headerSize)
        data.append(contentsOf: magic)
        data.append(version)
        data.append(flags.rawValue)
        data.append(contentType.rawValue)
        data.append(attachmentCount)
        data.append(recipientHash)
        data.append(ephemeralPublicKey)
        assert(data.count == SRC201Constants.headerSize)
        return data
    }

    /// Encode complete message to bytes
    public func encode() -> Data {
        var data = Data()

        // Header (72 bytes)
        data.append(headerBytes)

        // Nonce (24 bytes)
        data.append(nonce)

        // Payload length (2 bytes, big-endian)
        let length = UInt16(encryptedPayload.count).bigEndian
        withUnsafeBytes(of: length) { data.append(contentsOf: $0) }

        // Encrypted payload
        data.append(encryptedPayload)

        // Auth tag (16 bytes)
        data.append(authTag)

        // Optional: Reply-to (32 bytes)
        if flags.contains(.hasReplyTo), let replyTo = replyTo,
           let replyData = Data(hexString: replyTo) {
            data.append(replyData)
        }

        // Optional: Timestamp (8 bytes)
        if flags.contains(.hasTimestamp), let ts = timestamp {
            var tsBytes = ts.bigEndian
            withUnsafeBytes(of: &tsBytes) { data.append(contentsOf: $0) }
        }

        // Optional: Attachments
        if flags.contains(.hasAttachments) {
            for attachment in attachments {
                data.append(attachment.encode())
            }
        }

        return data
    }

    /// Decode message from bytes
    public static func decode(from data: Data) throws -> SRC201Message {
        guard data.count >= 114 else {
            throw SRC201Error.messageTooShort
        }

        // Verify magic
        let magic = Array(data.prefix(4))
        guard magic == SRC201Constants.magic else {
            throw SRC201Error.invalidMagic
        }

        let version = data[4]
        let flags = SRC201Flags(rawValue: data[5])

        guard let contentType = SRC201ContentType(rawValue: data[6]) else {
            throw SRC201Error.unknownContentType
        }

        let attachmentCount = data[7]
        let recipientHash = data[8..<40]
        let ephemeralPublicKey = data[40..<72]
        let nonce = data[72..<96]

        let payloadLength = UInt16(bigEndian: data[96..<98]
            .withUnsafeBytes { $0.load(as: UInt16.self) })

        guard data.count >= 114 + Int(payloadLength) else {
            throw SRC201Error.invalidPayloadLength
        }

        let encryptedPayload = data[98..<(98 + Int(payloadLength))]
        let authTag = data[(98 + Int(payloadLength))..<(114 + Int(payloadLength))]

        // Parse optional fields
        var offset = 114 + Int(payloadLength)
        var replyTo: String? = nil
        var timestamp: UInt64? = nil
        var attachments: [SRC201Attachment] = []

        if flags.contains(.hasReplyTo) && data.count >= offset + 32 {
            replyTo = data[offset..<(offset + 32)].hexString
            offset += 32
        }

        if flags.contains(.hasTimestamp) && data.count >= offset + 8 {
            timestamp = UInt64(bigEndian: data[offset..<(offset + 8)]
                .withUnsafeBytes { $0.load(as: UInt64.self) })
            offset += 8
        }

        if flags.contains(.hasAttachments) {
            for _ in 0..<attachmentCount {
                let (attachment, consumed) = try SRC201Attachment
                    .decode(from: data[offset...])
                attachments.append(attachment)
                offset += consumed
            }
        }

        return SRC201Message(
            magic: magic,
            version: version,
            flags: flags,
            contentType: contentType,
            attachmentCount: attachmentCount,
            recipientHash: Data(recipientHash),
            ephemeralPublicKey: Data(ephemeralPublicKey),
            nonce: Data(nonce),
            encryptedPayload: Data(encryptedPayload),
            authTag: Data(authTag),
            replyTo: replyTo,
            timestamp: timestamp,
            attachments: attachments
        )
    }
}

// MARK: - Attachment

public struct SRC201Attachment {
    public enum PartType: UInt8 {
        case inline = 0x01
        case external = 0x02
    }

    public enum ExternalProtocol: UInt8 {
        case ipfs = 0x01
        case arweave = 0x02
        case https = 0x03
    }

    public let partType: PartType
    public let contentType: SRC201ContentType
    public let blake3Hash: Data     // 32 bytes
    public let size: UInt32

    // For inline
    public let encryptedData: Data?
    public let tag: Data?           // 16 bytes

    // For external
    public let externalProtocol: ExternalProtocol?
    public let uri: String?
    public let encryptionKey: Data? // 32 bytes

    public func encode() -> Data {
        var data = Data()
        data.append(partType.rawValue)
        data.append(contentType.rawValue)
        data.append(blake3Hash)

        var sizeBytes = size.bigEndian
        withUnsafeBytes(of: &sizeBytes) { data.append(contentsOf: $0) }

        switch partType {
        case .inline:
            if let encryptedData = encryptedData {
                data.append(encryptedData)
            }
            if let tag = tag {
                data.append(tag)
            }
        case .external:
            if let proto = externalProtocol {
                data.append(proto.rawValue)
            }
            if let uri = uri {
                let uriBytes = uri.data(using: .ascii) ?? Data()
                data.append(UInt8(uriBytes.count))
                data.append(uriBytes)
            }
            if let key = encryptionKey {
                data.append(key)
            }
        }

        return data
    }

    public static func decode(from data: Data.SubSequence) throws
        -> (SRC201Attachment, Int) {
        // Implementation details...
        fatalError("TODO: Implement attachment decoding")
    }
}
```

## 11. Security Considerations

### 11.1 Threat Model

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                          THREAT MODEL (v1.1)                                 │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                              │
│  PROTECTED AGAINST:                                                          │
│  ──────────────────                                                          │
│  ✓ Message content interception (E2E encrypted)                             │
│  ✓ Recipient identification (registry-as-recipient pattern)                 │
│  ✓ Header tampering (AEAD with AAD authenticates headers)                   │
│  ✓ Message tampering (Poly1305 authentication tag)                          │
│  ✓ Replay attacks (nonce + expiry + per-sender nonce counter)               │
│  ✓ Cross-chain replay (chain_id in domain separator)                        │
│  ✓ Sender impersonation (Ed25519 transaction signatures)                    │
│  ✓ Registry forgery (meta-tx signature includes message hash)               │
│  ✓ Censorship (decentralized, no relay server)                              │
│  ✓ Spam/DoS (multi-layer anti-spam: rate limit + stake + filters)           │
│                                                                              │
│  METADATA VISIBLE ON-CHAIN:                                                  │
│  ────────────────────────────                                               │
│  ⚠ Sender address (from field - unavoidable)                                │

│  ⚠ Timestamp (block time)                                                   │
│  ⚠ That a message exists                                                    │
│                                                                              │
│  HIDDEN BY DESIGN:                                                           │
│  ─────────────────                                                          │
│  ✓ Recipient address (inside encrypted payload, only hash on-chain)         │
│  ✓ Message content                                                          │
│  ✓ Attachments                                                              │
│                                                                              │
│  NOT PROTECTED (KNOWN LIMITATIONS):                                          │
│  ───────────────────────────────────                                        │
│  ✗ Sender identity (visible unless using fresh addresses)                   │
│  ✗ Timing correlation (when messages are sent)                              │
│  ✗ Message count (how many messages an address sent)                        │
│  ✗ Key compromise (if Ed25519 private key is leaked)                        │
│                                                                              │
└─────────────────────────────────────────────────────────────────────────────┘
```

### 11.2 Implementation Requirements

```
SECURITY REQUIREMENTS FOR IMPLEMENTATION:

1. CRYPTOGRAPHIC LIBRARY SELECTION
   ────────────────────────────────
   • Use audited libraries only (libsodium, CryptoKit, ring)
   • NO custom cryptographic implementations
   • Constant-time operations for all key comparisons
   • Verify library versions against known vulnerabilities

2. KEY MANAGEMENT
   ────────────────
   • Store Ed25519 private keys in Secure Enclave (iOS) / TEE (Android)
   • Never log or transmit private keys
   • Clear key material from memory immediately after use (secureZero)
   • X25519 derived keys are ephemeral - generate fresh per message

3. NONCE HANDLING
   ───────────────
   • Always use cryptographically secure random for 24-byte nonce
   • Never reuse nonce with same key (XChaCha20's large nonce makes this safe)
   • Verify nonce uniqueness is guaranteed by your random source

4. AEAD USAGE
   ───────────
   • ALWAYS include header in AAD (bytes 0-71)
   • ALWAYS verify auth tag before processing decrypted data
   • Use same AAD construction for encrypt and decrypt
   • Reject ANY decryption failure with generic error (no oracle)

5. ERROR HANDLING (CRITICAL)
   ──────────────────────────
   • Same error message for "wrong key", "tampered", "corrupted"
   • Do NOT reveal which check failed
   • Rate limit decryption attempts per sender (prevent oracle attacks)
   • Log security events without sensitive data

6. INPUT VALIDATION
   ─────────────────
   • Validate magic bytes FIRST (fast reject)
   • Check message size before allocating memory
   • Verify all length fields for overflow
   • Reject unknown versions with clear error
   • Unknown flags should be IGNORED (forward compatibility)

7. SIGNATURE VERIFICATION
   ───────────────────────
   • Verify meta-tx signature BEFORE any processing
   • Check nonce is strictly greater than stored nonce
   • Check expiry is in the future
   • Domain separator prevents cross-chain replay
```

### 11.3 Timestamp Semantics

```
TIMESTAMP FIELD SEMANTICS:

The optional timestamp field (8 bytes, Unix epoch) is:
• SENDER-PROVIDED (not trustworthy for ordering)
• USEFUL FOR: UI display ("sent at"), message correlation
• NOT USEFUL FOR: canonical ordering (use block timestamp instead)

Clients SHOULD:
• Display sender timestamp if present (labeled as "sent")
• Use block timestamp for sorting/ordering
• Tolerate reasonable drift (±5 minutes)
• Reject timestamps far in the future (>1 hour ahead)

Block timestamp is authoritative for:
• Message ordering
• Expiry checks
• Rate limit calculations
```
