# SUM-721: Native NFT Standard

SUM-721 is the native NFT standard for SUM Chain, designed for both digital collectibles and certified document issuance.

## Overview

SUM-721 provides:
- **Native NFT support** - Built into the chain protocol, no smart contracts needed
- **Certified documents** - Non-transferable tokens for diplomas, licenses, certificates
- **Digital collectibles** - Transferable NFTs with royalty support
- **ERC-721 compatible API** - Familiar interface for developers

## Collection Types

### Standard Collectible
Transferable NFTs suitable for art, gaming items, and general digital assets.

```json
{
  "transferable": true,
  "burnable": true,
  "metadata_updatable": false,
  "owner_only_minting": true,
  "royalty_bps": 250,
  "max_supply": 10000
}
```

### Certified Document
Non-transferable, non-burnable tokens for official documents.

```json
{
  "transferable": false,
  "burnable": false,
  "metadata_updatable": false,
  "owner_only_minting": true,
  "royalty_bps": 0,
  "max_supply": 0
}
```

## NFT Operations

### CreateCollection
Create a new NFT collection.

| Field | Type | Description |
|-------|------|-------------|
| name | string | Collection name |
| symbol | string | Collection symbol (e.g., "ART") |
| description | string | Collection description |
| config | CollectionConfig | Collection configuration |
| base_uri | string (optional) | Base URI for token metadata |

### Mint
Mint a new token in a collection.

| Field | Type | Description |
|-------|------|-------------|
| collection_id | bytes32 | Target collection ID |
| to | Address | Recipient address |
| metadata | bytes | Token metadata (JSON or binary) |
| uri_type | string | "onchain", "ipfs", or "url" |
| uri_value | string (optional) | External URI |

### MintDocument
Mint a certified document NFT with issuer/subject information.

| Field | Type | Description |
|-------|------|-------------|
| collection_id | bytes32 | Target collection ID |
| to | Address | Document recipient |
| document_type | string | e.g., "degree", "license" |
| issuer | IssuerInfo | Issuing organization |
| subject | SubjectInfo | Document recipient info |
| document_hash | Hash | Hash of original document |
| issued_at | u64 | Issue timestamp |
| expires_at | u64 (optional) | Expiration timestamp |

### Transfer
Transfer a token to another address.

| Field | Type | Description |
|-------|------|-------------|
| collection_id | bytes32 | Collection ID |
| token_id | u64 | Token ID |
| to | Address | Recipient address |

### Approve
Approve an address to transfer a specific token.

| Field | Type | Description |
|-------|------|-------------|
| collection_id | bytes32 | Collection ID |
| token_id | u64 | Token ID |
| approved | Address (optional) | Approved address (None to revoke) |

### Burn
Destroy a token (if collection allows).

| Field | Type | Description |
|-------|------|-------------|
| collection_id | bytes32 | Collection ID |
| token_id | u64 | Token ID to burn |

### LockToken / UnlockToken
Lock or unlock a token to prevent transfers.

| Field | Type | Description |
|-------|------|-------------|
| collection_id | bytes32 | Collection ID |
| token_id | u64 | Token ID |

  "method": "nft_getCollection",
  "params": ["0x...collection_id..."],
  "id": 1

## CLI Commands

The `sumchain-wallet` CLI provides NFT query commands:

```bash
# Get collection info
sumchain-wallet nft-collection --id 0x...

# Get token info
sumchain-wallet nft-token --collection 0x... --token-id 42

# List tokens owned by an address
sumchain-wallet nft-list --owner SUM1abc...

# Get NFT balance
sumchain-wallet nft-balance --owner SUM1abc...

# Get token owner
sumchain-wallet nft-owner --collection 0x... --token-id 42
```

## Document Certification Workflow

### 1. Create a Document Collection

An institution creates a collection for their certificates:

```
Collection: "MIT Computer Science Degrees"
Symbol: "MIT-CS"
Config: certified_document (non-transferable, non-burnable)
```

### 2. Issue a Certificate

When a student graduates:

```
Mint Document NFT:
- Recipient: Student's SUM Chain address
- Document Type: "degree"
- Issuer: MIT (verified on-chain address)
- Subject: John Doe (student info)
- Document Hash: SHA256 of the PDF diploma
- Expiration: None (permanent)
```

### 3. Verify a Certificate

Anyone can verify:

1. Check the NFT exists via `nft_tokenExists`
2. Verify the issuer address matches MIT's official address
3. Compare the document hash with the original PDF
4. Check expiration status (for licenses)

## Security Considerations

1. **Collection ownership** - Only the collection owner can mint (if `owner_only_minting` is set)
2. **Transfer restrictions** - Certified documents cannot be transferred
3. **Burn restrictions** - Permanent documents cannot be burned
4. **Token locking** - Owners can lock tokens to prevent accidental transfers
5. **Approval system** - Standard ERC-721 approval for delegated transfers

## Differences from ERC-721

| Feature | ERC-721 | SUM-721 |
|---------|---------|---------|
| Implementation | Smart contract | Native protocol |
| Gas costs | Variable | Fixed, lower |
| Operator approval | Yes | Planned |
| Royalties | EIP-2981 extension | Built-in |
| Document mode | Not supported | Native support |
| Token locking | Not supported | Native support |
