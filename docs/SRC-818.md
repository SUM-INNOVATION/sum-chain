# SRC-818: Course Offering Token

> **Status:** Phase 0 draft — design baseline, pre-legal-review. No wire types, no executor, no activation gate exist yet. This document specifies intended behavior for review only.

## Overview

SRC-818 defines a **Course Offering** — one live class instance of a course: a specific term and section, taught by specific instructor(s), with an enrolled roster, assignments/exams, submissions, and grades.

**Core product invariant — stated up front:**

> **`SubmitAssignment` and `SubmitExam` are chain transactions. The actual submitted content lives in SNIP. The chain records only a SNIP reference + a content commitment + the submission timestamp/height + the attempt number.** Raw coursework is never placed on-chain.

The same principle governs grading: `GradeSubmission` is a chain transaction recording a **grade commitment** and an encrypted-feedback `SnipRef` — never the raw grade or feedback text.

Every SRC-818 offering **must** reference a non-Archived/Deprecated [SRC-817](SRC-817.md) catalog entry via `catalog_id` (one catalog → many offerings). Authorization is delegated: students/enrollment via **SRC-812**, instructors/TAs via **SRC-882**. The shared glossary, privacy/SNIP model, `ContentAccessPolicy` definition, transaction envelope, fee/nonce open question, and consolidated legal/product questions are in the companion document [SRC-81X-EDUCATION-SUITE.md](SRC-81X-EDUCATION-SUITE.md).

## Design Principles

Privacy non-negotiables (v1, locked baseline):

1. **No raw submissions on-chain** — only `content_ref` (SnipRef) + `content_commitment`.
2. **No raw grades on-chain** — only `grade_commitment` + encrypted `feedback_ref`. Authoritative transcripts/credentials are the job of SRC-810/SRC-811, **not** SRC-818.
3. **No answer keys on-chain** — only `answer_key_commitment`; plaintext released via SNIP post-close.
4. **No student PII on-chain** — names/IDs never appear in any form.
5. **`student_commitment`, not raw address**, is the default identifier in records and indexes. Raw-address indexes are excluded from v1 unless explicitly signed off by legal/product (off-chain private indexer preferred).
6. **Mandatory `ContentAccessPolicy`** for every `SnipRef` in chain state. No dangling content refs.
7. **No unbounded `Vec` in primary records.** Content items, assessments, enrollment links, submissions, and grades are separate CF rows; the `Offering` primary record holds only counters/roots.

## Data Model

### `Offering` (primary record — counters/roots only)

Stored one-per `offering_id` in `edu_offerings`.

```text
Offering {
    offering_id:       [u8;32]   // see Commitment Generation
    catalog_id:        [u8;32]   // REQUIRED ref to SRC-817 (non-Archived/Deprecated)
    term:              String    // public coordinate (e.g. "2026FA")
    section:           String    // public coordinate (e.g. "A")
    owner:             Address    // department/institution admin (offering controller)
    instructor_count:  u32
    instructor_root:   [u8;32]    // root over InstructorBinding child rows
    content_count:     u32
    content_root:      [u8;32]    // root over ContentItem child rows
    assessment_count:  u32
    assessment_root:   [u8;32]    // root over AssessmentRef child rows
    enrollment_count:  u32        // count only — never a roster
    enrollment_root:   [u8;32]    // root over EnrollmentLink child rows
    status:            OfferingStatus
    policy_id:         Option<[u8;32]>   // optional SRC-803 hook, off by default v1
    created_at_height: BlockHeight
    updated_at_height: BlockHeight
    nonce:             u64
}

OfferingStatus { Draft=0, Active=1, EnrollmentClosed=2, Completed=3, Archived=4, Suspended=5, Cancelled=6 }
CourseRole     { InstitutionAdmin, Instructor, TeachingAssistant, Grader, Student, Auditor }
AssessmentKind { Assignment=0, Exam=1, Quiz=2, Project=3 }
SubmissionStatus { Submitted=0, Graded=1, Resubmitted=2, Voided=3 }
GradeStatus      { Provisional=0, Finalized=1, Revised=2 }
AssessmentStatus { Draft=0, Open=1, Closed=2, Graded=3 }
ContentKind      { Syllabus, LectureMaterial, Reading, Resource, Other }
```

### Child records (separate CF rows — bounded-collection rule)

```text
InstructorBinding {           // CF edu_instructor_bindings, key offering_id || actor
    actor:                      Address
    role:                       CourseRole
    employment_credential_ref:  [u8;32]   // → SRC-882 EmploymentCredential
}

ContentItem {                 // CF edu_content_items, key offering_id || content_id
    content_id:        [u8;32]
    kind:              ContentKind
    snip_ref:          SnipRef
    content_commitment:[u8;32]
    access_policy:     ContentAccessPolicy   // mandatory
}

AssessmentRef {               // CF edu_assessments, key offering_id || assessment_id
    assessment_id:        [u8;32]
    kind:                 AssessmentKind
    instructions_ref:     SnipRef
    instructions_access:  ContentAccessPolicy           // mandatory
    spec_commitment:      [u8;32]   // binds due/attempts/weight, revealed off-chain
    opens_at:             Timestamp
    due_at:               Timestamp
    max_attempts:         u16
    weight_bps:           u16       // plaintext: needed for on-chain window/attempt checks
    answer_key_commitment:Option<[u8;32]>
    answer_key_access:    Option<ContentAccessPolicy>   // mandatory if key present
    status:               AssessmentStatus
}

EnrollmentLink {              // CF edu_enrollment_links, key offering_id || student_commitment
    student_commitment: [u8;32]   // NOT raw address
    enrollment_ref:     [u8;32]   // → SRC-812 (DocClass subcode 812)
    linked_at_height:   BlockHeight
}

SubmissionRecord {            // CF edu_submissions,
                              // key offering_id || assessment_id || student_commitment || attempt
    submission_id:      [u8;32]
    student_commitment: [u8;32]
    attempt:            u16
    content_ref:        SnipRef        // student-OWNED SNIP object
    content_commitment: [u8;32]
    submitted_at:       Timestamp
    submitted_at_height:BlockHeight
    late:               bool
    enrollment_ref:     [u8;32]        // SRC-812 credential used to authorize (audit)
    access_policy:      ContentAccessPolicy   // student + instructor + assigned graders
    status:             SubmissionStatus
}

GradeRecord {                 // CF edu_grades,
                              // key offering_id || assessment_id || student_commitment
    grade_id:          [u8;32]
    submission_id:     [u8;32]
    grade_commitment:  [u8;32]         // NEVER the raw grade
    feedback_ref:      Option<SnipRef>
    feedback_access:   Option<ContentAccessPolicy>   // student + instructor/grader only
    grader:            Address
    grader_role:       CourseRole
    graded_at_height:  BlockHeight
    status:            GradeStatus
    finalized:         bool
}
```

`SnipRef`, `ContentAccessPolicy`, `AccessAudience` canonical definitions: companion doc.

## `ContentAccessPolicy` application matrix

Every SnipRef above pairs with a policy. Default policies:

| Object | `AccessAudience` | opens_at | closes_at | grace_until | revoke_on_course_archive |
|---|---|---|---|---|---|
| Syllabus / lecture materials | `EnrolledStudents` | publish | none | n/a | true |
| Assignment instructions | `EnrolledStudents` | publish | none | n/a | true |
| Exam content | `EnrolledStudents` | exam open | exam close | optional | true |
| Answer key | `StaffOnly` → (post-close) `EnrolledStudents` | (key set) / (assessment close) | none | n/a | true |
| Submission | `IndividualStudent(student_commitment)` + instructor + assigned graders | submit | none | n/a | per retention policy |
| Feedback / grade detail | `IndividualStudent(student_commitment)` + instructor/grader | grade publish | none | n/a | per retention policy |

The chain stores the policy/schedule + commitments + refs. **SNIP enforces** actual private object access (encrypted ACL / key bundles deliver decryption to exactly the named audience within the time window). The chain never holds decryption keys.

## SNIP object-ownership model

- The **instructor controls publication and access policy** for course content (not "owns" it).
- **SNIP object ownership varies by object type:** syllabus / lecture materials / assignment instructions / exam content / answer keys → owned by **institution or instructor**; feedback / grade detail → owned by **instructor/grader**.
- **Submissions: the student owns/controls the submitted SNIP object.** `SubmitAssignment`/`SubmitExam` *grants* scoped access to the instructor and assigned graders per the course `ContentAccessPolicy`; it does **not** transfer ownership.

## Operations

Delivered as `EducationTxData { standard: CourseOffering, operation, data, recipient }`. Codes stable/append-only.

| Code | Operation | Actor | Key preconditions | Effect |
|---|---|---|---|---|
| 0 | `CreateOffering` | InstitutionAdmin | `sender==owner`; `catalog_id` resolves to Active/Deprecated SRC-817 entry | New `Offering` (Draft) |
| 1 | `UpdateOffering` | Admin / Instructor | status ∈ {Draft, Active} | Metadata edits |
| 2 | `PublishContent` | Instructor | SRC-882 teaching role; status ∈ {Draft, Active} | Add/update `ContentItem` + access policy |
| 3 | `AddAssessment` | Instructor | SRC-882 teaching role | Add `AssessmentRef` + access policies |
| 4 | `UpdateAssessment` | Instructor | assessment status ∈ {Draft, Open} | Adjust window/weight/policy |
| 5 | `OpenEnrollment` | InstitutionAdmin | status ∈ {Draft, Active} | Accept enrollment links |
| 6 | `CloseEnrollment` | InstitutionAdmin | status == Active | → `EnrollmentClosed` |
| 7 | `LinkEnrollment` | Student (self) / Admin | SRC-812 credential valid + subject-bound to sender | Bind `enrollment_ref`; store `student_commitment` |
| 8 | **`SubmitAssignment`** | **Student** | see pipeline below | New `SubmissionRecord` |
| 9 | **`SubmitExam`** | **Student** | pipeline below; `kind==Exam`, within exam window | New `SubmissionRecord` |
| 10 | `GradeSubmission` | Instructor/TA/Grader | SRC-882 grading role; submission exists | New `GradeRecord` (Provisional) |
| 11 | `FinalizeGrade` | Instructor | grade status == Provisional | `Provisional → Finalized` |
| 12 | `FinalizeCourse` | InstitutionAdmin | all grades Finalized | → `Completed` |
| 13 | `ArchiveOffering` | InstitutionAdmin | status == Completed | → `Archived`; trigger `revoke_on_course_archive` policies |
| 14 | `SuspendOffering` / `CancelOffering` | InstitutionAdmin | suspend reversible; cancel terminal | → `Suspended` / `Cancelled` |

### `SubmitAssignment` / `SubmitExam` validation pipeline (normative)

Ordered; fail-stop. **Fee/nonce on a failed step is an open chain-consistency question — companion doc §Fee/Nonce — and is NOT decided here.**

1. Education activation gate open, else reject.
2. Offering exists, `status ∈ {Active, EnrollmentClosed}`; rejected in `Draft`, `Suspended`, `Cancelled`, `Archived`. (`EnrollmentClosed` blocks only *new enrollment links* — coursework submission and grading remain open.)
3. Assessment exists under offering; `kind` matches the operation; `status == Open`; `opens_at ≤ now ≤ due_at`, or `due_at < now ≤ grace_until` → set `late = true` (accept-and-flag; penalty computed off-chain).
4. `sender` resolves to the `student_commitment` of an existing `EnrollmentLink` on this offering.
5. SRC-812 credential (`enrollment_ref`) valid: exists, non-revoked (SRC-805), in validity window, subject-bound to `sender`, scoped to this offering/term.
6. Attempt count for `(offering, assessment, student_commitment)` `< max_attempts`.
7. Transaction `data` carries only `content_ref` + `content_commitment`; validator rejects raw/oversize payloads (tight max on `data`).
8. Write `SubmissionRecord`, update child-row indexes, increment counters. Fee/nonce per the open chain-consistency decision.

Distinct failure classes (gate-closed / offering-not-active / window-closed / not-enrolled / attempts-exhausted / wrong-sender / raw-payload-rejected) so clients can react precisely.

## Lifecycle

```
Draft ─Open─► Active ─Close─► EnrollmentClosed ─Finalize─► Completed ─Archive─► Archived
  │            │  ▲                  │                                   (read-only,
  │            │  └──reopen(admin)───┘                                    terminal)
  │            ▼
  └─► Cancelled (terminal)     Suspended ──resume──► (prior state)
```

| From | To | Operation | Notes |
|---|---|---|---|
| Draft | Active | `OpenEnrollment` | Enrollment accepting |
| Active | EnrollmentClosed | `CloseEnrollment` | Submissions/grading still allowed |
| EnrollmentClosed | Active | `OpenEnrollment` | Reopen pre-Completion |
| EnrollmentClosed | Completed | `FinalizeCourse` | Requires all grades Finalized |
| Completed | Archived | `ArchiveOffering` | Triggers `revoke_on_course_archive` |
| {Active, EnrollmentClosed} | Suspended | `SuspendOffering` | Blocks mutating ops; resumes to prior state |
| {Draft, Active} | Cancelled | `CancelOffering` | Terminal; records retained for audit |

Submissions/grading occur in `Active` and `EnrollmentClosed`. `Archived`/`Cancelled` are terminal/read-only.

## Commitment Generation

```text
offering_id        = BLAKE3(b"SRC818-OFFERING:v1:" || catalog_id || term || section || creator || nonce_le)
student_commitment = BLAKE3(b"SRC818-STUDENT:v1:"  || subject || offering_id || salt)
content_commitment = BLAKE3(b"SRC818-SUBMISSION:v1:" || offering_id || assessment_id || student_commitment || attempt_le || work_hash || salt)
grade_commitment   = BLAKE3(b"SRC818-GRADE:v1:"    || offering_id || assessment_id || student_commitment || grade_value || salt)
spec_commitment    = BLAKE3(b"SRC818-SPEC:v1:"     || assessment_id || due_le || max_attempts_le || weight_le || salt)
```

Because `submitted_at_height`, `due_at`, and attempt counts are on-chain, **lateness and attempt-limit compliance are publicly verifiable without exposing the work**.

## Transaction Format

```text
EducationTxData {
    standard:  EducationStandard::CourseOffering,
    operation: u16,        // SRC-818 operation code
    data:      Vec<u8>,    // bincode of the operation struct
    recipient: Address,    // soulbound target where relevant (else owner)
}
```

`Education` = `TxPayload` variant 22 / `TxType = 22` (append-only). No offering/submission/grade/enrollment record is transferable — soulbound by default; no transfer operation exists.

## Authorization

| Operation | InstitutionAdmin | Instructor | TA/Grader | Student | Source of authority |
|---|---|---|---|---|---|
| Create/Update/Open-Close-Enroll/Finalize/Archive/Suspend/Cancel | ✅ `sender==owner` | — | — | — | SRC-818 offering owner |
| PublishContent / Add-UpdateAssessment | ✅ | ✅ | — | — | SRC-882 (teaching role) |
| LinkEnrollment | ✅ (on behalf) | — | — | ✅ (self) | SRC-812 credential |
| SubmitAssignment / SubmitExam | — | — | — | ✅ (self only) | SRC-812 + sender==student_commitment |
| GradeSubmission | ✅ | ✅ | ✅ | — | SRC-882 (grading role) |
| FinalizeGrade | ✅ | ✅ | — | — | SRC-882 |

Soulbound by default; controller hand-off (if ever) is a future admin op, never a market transfer.

## Security Considerations

1. **Submission spam / DoS.** Mitigated by attempt limits + enrollment binding; residual risk depends on the open fee/nonce decision. If zero-fee-on-failure is chosen, mempool admission (Phase 3) must pre-check gate/offering-active/enrollment/window/attempts.
2. **Enrollment-binding integrity.** Submission requires a valid, non-revoked, subject-bound SRC-812 credential scoped to the offering.
3. **Late/attempt provability.** On-chain heights + counts make compliance publicly verifiable without exposing work.
4. **SNIP enforcement boundary.** Chain stores policy/commitments/refs only; confidentiality depends on SNIP encrypting to the policy-named audience. The chain cannot leak content it never holds.
5. **Append-only discipline.** No CF/enum reorder; additive evolution only.

## Example Usage (illustrative, design-level)

```text
1. CreateOffering { catalog_id: <SRC-817 CS101>, term:"2026FA", section:"A" } → Draft
2. PublishContent { ContentItem(Syllabus, SnipRef, policy=EnrolledStudents) }
3. AddAssessment { kind=Assignment, instructions_ref, opens_at, due_at, max_attempts:2 }
4. OpenEnrollment ; student LinkEnrollment { enrollment_ref: <SRC-812 cred> }
5. Student SubmitAssignment { content_ref: <SNIP object>, content_commitment }
   → SubmissionRecord (chain tx; content stays in SNIP)
6. Instructor GradeSubmission { grade_commitment, feedback_ref (encrypted) } → Provisional
7. FinalizeGrade → FinalizeCourse → ArchiveOffering (revoke_on_course_archive fires)
```
