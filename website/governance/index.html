Governance

Governance at the consensus level.

Group control on sum chain is not a smart contract pattern. Policy accounts are enforced directly by the state executor, so shared ownership carries no contract risk.

Policy accounts

A policy account is an address controlled by a group. Members carry optional voting weights, and a policy defines the approval threshold for each class of action: unanimous, majority, percentage, weighted, or custom.

The rules run in the state executor, not in a smart contract. There is no contract code to exploit, approvals carry nonce-based replay protection, and unknown action types fail closed until explicitly configured.

Profiles

Pre-configured policies cover the common cases:

ConservativeUnanimous approval for ownership transfers
CompanyMajority for governance, unanimous for asset transfers
DAOToken-weighted voting
TrustConservative thresholds with fiduciary requirements
PersonalSimple majority for shared accounts

Proposals

Any member can submit a proposal containing the action to execute, such as transferring an NFT the group owns. Members approve; once the threshold for that action class is met, the proposal executes and the account's nonce increments. Three people can jointly own a tokenized house and be certain it cannot be sold without all three signatures.

Validators and epochs

Block production runs under Proof of Authority with round-robin rotation. The active validator set is recomputed every epoch of 14,400 blocks using stake-weighted selection. Missed blocks are tracked per validator, misbehavior is slashed, and repeat offenders can be tombstoned permanently.
