Run a node

Run the chain yourself.

The node is a single Rust binary. Building from source is one command, mainnet runs on modest hardware, and everything is inspectable over JSON-RPC.

Build

The node is a single binary built from source with the stable Rust toolchain (1.85 is pinned in the repository). There are no C or C++ dependencies to fight with.

git clone https://github.com/SUM-INNOVATION/sum-chain
cd sum-chain
cargo build --release

# Binaries in ./target/release/
# - sumchain         (full node)
# - sumchain-wallet  (CLI wallet)

Configure

cat > config.toml << EOF
[node]
genesis = "genesis.json"
data_dir = "data"
validator_key = "validator-key.json"

[network]
listen_addr = "/ip4/0.0.0.0/tcp/9933"

[rpc]
addr = "127.0.0.1:8545"
EOF

Run

# Generate a key
./target/release/sumchain keygen --output validator-key.json

# Start the node
./target/release/sumchain run \
  --config config.toml \
  --genesis genesis.json

Monitor

Every node exposes operator endpoints over JSON-RPC:

# Liveness, height, peers, sync state
{"jsonrpc":"2.0","method":"health","params":[],"id":1}

# Version, peer id, uptime
{"jsonrpc":"2.0","method":"node_info","params":[],"id":1}

# Prometheus-style metrics
{"jsonrpc":"2.0","method":"get_metrics","params":[],"id":1}

Joining the validator set

Consensus is Proof of Authority with round-robin proposer rotation. The active set is recomputed each epoch with stake-weighted selection, and validators earn every fee in the blocks they propose. If you want to operate a validator, write to hello@suminnovation.xyz.

Local development

For client integration without a full testnet, the repository ships a self-bootstrapping single-validator Docker preset on the snip-local-mirror-preset branch. It generates a disposable key on first boot and exposes RPC on localhost:8545.
