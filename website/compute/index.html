Compute

Compute you can verify.

OmniNode settles zero-knowledge attestations of AI inference on sum chain. Pay once for the compute; the proof that it ran correctly stays on chain permanently.

Inference attestations

OmniNode runs AI inference off chain and verifies it with zero-knowledge proofs. What lands on sum chain is an attestation: a verifier-signed digest binding a session identifier to four content hashes, covering the model, the input manifest, the response, and the proof.

InferenceAttestationDigest {
    session_id:    String,
    model_hash:    [u8; 32],
    manifest_root: [u8; 32],
    response_hash: [u8; 32],
    proof_root:    [u8; 32],
}

The verifier signs twice with the same Ed25519 key: the outer transaction through the chain's standard signing path, and the inner digest under OmniNode's dedicated signing domain. The executor enforces that sender and verifier are the same key.

Permanence

One attestation per session and verifier pair, forever. Duplicates are rejected both at mempool admission and at executor dispatch, and the record survives for the life of the chain. Anyone can later check that a given inference happened, was verified, and produced the claimed output.

Query it

# Attestation for a session
{"jsonrpc":"2.0","method":"sum_getInferenceAttestation",
 "params":["session-id"],"id":1}

# Enumerate attestations
{"jsonrpc":"2.0","method":"sum_listInferenceAttestations","params":[],"id":1}

# Status check
{"jsonrpc":"2.0","method":"sum_getInferenceAttestationStatus",
 "params":["session-id"],"id":1}

Status

The v1 wire format is frozen and merged, with byte-level fixture tests locking the format against drift. Mainnet activation is gated by the omninode_enabled_from_height chain parameter and follows a coordinated validator upgrade.

OmniNode itself is live at omninode.suminnovation.xyz.
