Wallet

Your keys, two ways.

Use Koppa from the browser with SUMailet or from the terminal with the CLI wallet. Both are non-custodial and speak to the same chain.

SUMailet, in the browser

SUMailet is the web wallet: send and receive Koppa from any browser with nothing to install. It is non-custodial; keys stay with you.

The CLI wallet

Built alongside the node from the same repository:

git clone https://github.com/SUM-INNOVATION/sum-chain
cd sum-chain && cargo build --release
# Create a wallet
sumchain keygen --output my-wallet.json

# Check a balance
sumchain-wallet balance --key my-wallet.json

# Send Koppa
sumchain-wallet transfer \
  --key my-wallet.json \
  --to RECIPIENT_ADDRESS \
  --amount 10.5

Transfers confirm in the next block and are final after six confirmations, roughly eighteen seconds. The typical fee is 0.001 Ϙ.

Addresses and keys

SignaturesEd25519
AddressBlake3(pubkey)[12..32]
Display formatsBase58 with checksum, or 0x-prefixed hex
Messaging keysX25519, derived from the same identity

The same keypair that holds Koppa can receive SRC-201 encrypted messages and hold document credentials; one identity across the whole protocol.

In development

A TypeScript SDK and native mobile wallets are in progress. Until then, the JSON-RPC API is stable and fully documented for direct integration.
